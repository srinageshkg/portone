ACID

  • Atomicity:transaction is treated as a single, indivisible unit of work Atomicity is often achieved using techniques like transaction logs and write-ahead logs
  • Consistency: guarantees that a transaction brings the database from one consistent state to another consistent state
  • Isolation:prevents interference or data corruption when multiple transactions are executed concurrently
  • Durability: guarantees that once a transaction is committed, its changes to the database are permanent and will survive any subsequent system failures
